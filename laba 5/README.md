# Лабораторная работа № 5
___
При решении был создан вспомогательный класс **Metod** для проверок.
## Задание 1. Шаблоны
```
В класс Дробь, добавить интерфейс на два метода: получение вещественного значения, установка
числителя и установка знаменателя.
Сгенерировать такую версию дроби, которая будет кэшировать вычисление вещественного
значения.
Если раннее в вашем варианте не было Дроби, то создайте сущность Дробь со следующими
особенностями:
• Имеет числитель: целое число
• Имеет знаменатель: целое число
• Дробь может быть создана с указанием числителя и знаменателя
• Может вернуть строковое представление вида “числитель/знаменатель”
• Необходимо корректно обрабатывать отрицательные значения. Учтите, что знаменатель не может
быть отрицательным.
• Переопределите метод сравнения объектов по состоянию таким образом, чтобы две дроби
считались одинаковыми тогда, когда у них одинаковые значения числителя и знаменателя.
```
Для решения задания были созданы:
- Интерфейс **FracInter**.
- Класс **Fraction**, который использует интерфейс **FracInter**.
- Класс **CachedFraction**, который наследуется от **Fraction**.
---
### Поговорим о каждом отдельно.  

### interface FracInter :  
В интерфейсе описано 3 метода.  
```
double getValue(); //результат вычисления числа
void setNumer(int numer); //внесение знаменателя
void setDenom(int denom); //внесение числителя
```

### Fraction implements FracInter :  

- **private int numer** - поле, отвечающее за числитель.  
- **private int denom** - поле, отвечает за знаменатель.  
- Геттеры **getNumer()**,**getDenom()** - методы для получения значений числителя и знаменателя.  
- Сеттер **setNumer(int numer)** - переопределённый метод для внесения числителя.  
- Сеттер **setDenom(int denom)** - переопределённый метод для внесения знаменателя. В нём есть проверка, чтобы внесённый параметр не был 0.
```java
if (denom == 0) {
    throw new IllegalStateException("Знаменатель не может равняться 0"); //ошибка, если значение равно 0
}

if (denom < 0) { //если было отрицательно, то меняем на на положительное. Так как знаменятель не может быть отрицательным
    this.numer = -this.numer; //переносим этот "минус" в числитель
    denom = -denom; // меняем знаменатель на положительный 
}
this.denom = denom; //вводим значение в поле
```
- Конструктор **Fraction(int numer, int denom)** - Для создания дроби с 2 параметрами. Где также есть проверка, 
что знаменатель не 0 и правильно обрабатывались отрицательные знаменатели.
- boolean **equals(Object obj)** - переопределённый метод для сравнения дробей
```java
if (this == obj) {
    return true; //ссылочное равенство
}
if (!(obj instanceof Fraction)) { //Проверка: объект obj аявляется экземпляром класса Fraction или его наследника?
    return false;
}
Fraction other = (Fraction) obj; //если до этого не было ошибки,то создаём сушьность дробь с сылкой на проверяемый объект
if (other.numer==numer && other.denom==denom) { //если числитель и знаменатель дроби равен знаменятелю и числителю другой дроби
    return true;
} else {
    return false;
}
```
- String **toString()** - переопределённый метод для строкового представления дроби.  
- int **hashCode()** - переопределённый метод для вычисления общего хэш-кода для числителя и знаменателя.  
```java
return Objects.hash(numer, denom); // hash = 31*numer + denom
```
### CachedFrac  extends Fraction:
- private **Double cachedV** - поле, хранящее последнее вычисленное вещественное значение дроби.При первом создании дроби равно null  
- **CachedFrac(int numer, int denom)** - конструктор для создания дроби. Использует 
конструктор родительского класса Fraction.
- **setNumer(int numer)** - переопределённый сеттер для внесения числителя в дробь. Использует сеттер родительского класса.  
- **setDenom(int denom)** - переопределённый сеттер для внесения знаменателя в дробь. Также используем метод из родительского класса.  
- **double getValue()** - переопределённый метод для получения вещественного числа.  
```java
if (cachedV == null) { //если до этого вещественное значение не было вычислено
    System.out.println("Вычисляю");
    cachedV = super.getValue(); //вычисляем через родительский метод
}
return cachedV; // возвращаем значение
```
